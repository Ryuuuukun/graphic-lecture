# Обратная трассировка лучей с использованием полигональных моделей

## Обзор задания

### Алгоритм

При трассировке лучей используются триангулированные полигональные модели, поскольку треугольник всегда лежит в одной плоскости.

Задача трассировки лучей с использованием полигональных моделей сводится к задаче поиска пересечения луча с треугольником. Полученная задача может быть разбита на две подзадачи:
- Поиск точки пересечение луча с плоскостью, содержащую данный треугольник.
- Определить, попадает ли полученная точка пересечения в треугольник.

Первая подзадача имеет тривиальное и очевидное решение. Для второй подзадачи могут быть разные решения. В текущем проекте используется алгоритм основанный на свойстве скалярного произведения: скалярное произведение единичного радиус вектора OP на единичный орт оси Ox больше нуля когда точка P лежит справа от оси Oy и меньше нуля когда точка P лежит слева от оси Oy.

Для простоты алгоритм можно рассмотреть на двумерной плоскости. Пусть дан треугольник ABC и точка пересечения P. Рассмотрим направлящие вектора (образы оси Oy, на рисунке отмечены фиолетовым) для каждой вершины треугольника в порядке обхода по часовой стрелке (выбор направление не влияет на результат, но для каждой вершины он должен быть одинаковый). Для каждого направляющего вектора рассмотрим перпендикулярный ему (образ оси Ox, на рисунке отмечены зеленым). Рассматривая скалярное произведение векторов AP, BP, CP и A', B', C' соотвественно получим набор знаков. Если все знаки одинаковые, то точка P принадлежит треугольнику, в противном случае, если знаки отличаются, не принадлежит.

Заметный плюс алгоритма, при кэшировании перпендикуляров можно добится хорошей производительности. Значительным минусом алгоритма является потребление памяти, поскольку на каждый треугольник необходимо хранить еще три вектора, что сильно увеличивает потребление памяти.

### Поддержка Wavefront OBJ

Программа поддерживает ограниченную загрузку моделей в формате wavefront .obj. Перед экспортом в `blender` необходимо установить следущую конфигурацию экспорта:

- [ ] UV Coordinates
- [x] Normals
- [ ] Colors
- [ ] Curves as NURBS
- [x] Triangulated Mesh
- [x] Apply Modifiers
- [x] Apply Transform
- [ ] Materials
- [ ] Animation

### Структура проекта

- `component` - вспомогательные компоненты.
- `geometry` - реализация алгоритма, обработки полигональных моделей и их хранения.
- `linear` - линейная алгебра, ограничивающие объемы, кватернионы.
- `postprocess` - постобработка: алгоритм анти алиазинга FXAA (вместо суперсэмплинга), виньетка.
- `render` - реализация устройства рендеринга и алгоритмов
- `scene` - реализация сцены и камеры
- `utility` - утилиты для работы с контекстом библиотеки SDL2, а также импорта моделей.
